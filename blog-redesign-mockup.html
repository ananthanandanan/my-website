Your Users Will Break Your App (and That's a Good Thing)

Dec 29, 2024 6 min read
Real users don't behave like test cases. They use your app at 2am on a slow connection, paste weird unicode into your text fields, and click buttons three times because the first two felt unresponsive. And every one of those moments is signal.

When I shipped the first version of QuestCraft's assessment builder, I had a comprehensive test suite. Unit tests, integration tests, even a few end-to-end tests. I was confident. Then real teachers started using it.

The First Week

Within 48 hours, I had a Sentry inbox full of errors I'd never imagined. A teacher in rural Kerala was using it on a 2G connection. Another was pasting content directly from a Google Doc, which introduced U+202F — a narrow no-break space — into a field that expected plain ASCII.

Production is the only environment that matters. Everything else is rehearsal.

What the Errors Taught Me

Each bug was a window into a use case I hadn't modeled. Here's a simplified version of the Unicode issue:

python 
# Before: naive validation
def validate_question_text(text: str) -> bool:
    return text.strip().isascii()

# After: Unicode-aware sanitization
import unicodedata

def sanitize_question_text(text: str) -> str:
    # Normalize to NFC form, strip weird spaces
    normalized = unicodedata.normalize("NFC", text)
    return " ".join(normalized.split())

The Slow Connection Problem

The 2G user taught me something more important than any error log. My UI had no meaningful loading states. Buttons would sit there, unresponsive, because the network call was in flight. Users clicked again. And again. Creating duplicate records.

The fix was boring — disable buttons on submission, add optimistic UI updates. But I never would have prioritized it without a real user showing me the problem. This is the gift of production traffic.

Turning Breakage into Signal

Now I treat every unexpected error as a user interview. Sentry's context breadcrumbs tell me what the user did before the crash. Session replays (via PostHog) show me exactly where people get confused. These aren't bugs — they're product research.

Things I believe
